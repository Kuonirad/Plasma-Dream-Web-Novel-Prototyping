## 2026-04-19 - [Critical] Unhandled NaN / Out-of-Bounds Inputs in Mathematical Controllers
**Vulnerability:** The Python `BayesianBetaController.update` method accepted unvalidated floats. If passed `NaN` or non-numeric inputs from spectral analysis, these propagate through mathematical functions like `np.clip` converting floats to ints, causing a `ValueError` (e.g., `cannot convert float NaN to integer`), leading to Denial of Service / controller crashes.
**Learning:** Mathematical models (especially those using probabilities or bounds clipping) are highly susceptible to poisoning via non-finite or invalid types. In Python, `float('nan')` interacts poorly with `np.clip` when cast to integer.
**Prevention:** Implement strict input sanitization on all sensor data entry points. Specifically, typecast to float, handle `NaN` gracefully (e.g., substituting with a safe default like `0.0`), and strictly bound inputs using `np.clip(val, min, max)` before they enter computational state vectors.
